data-url - your search API endpoint e.g. server.php, https://mywebsite.com/api/search

data-storage - weither to store results or not, default to true, value must be true or false only

data-spellchecker - weither to use spellchecker mechanism or not, default to true, value must be true or false only

data-recentsearches - weither to use the recent searches mechanism or not, default to true, value must be true or false only

data-db - custom db name

data-modify-results - a function name where raw data will be pass on before render e.g.

data-template - this is essential, basically, this is just a template for the results, where dollar sign ($) + word will represent the result object from the attribute data-key

Template Data Attributes

data-keys - object keys that the plugin expect from the api response e.g. 

		
[{ key1 : 'key1', key2 : 'key2', key3 : 'key3' }];

data-orderkey - by default, the plugin sort the results by the word closest to the keyword base on the first key on the data-keys, yet you can also define the order key by specifying it on data-orderkey e.g. 

		
data-orderkey="key2"

By default, storage will reset in 5 days or when you specify a version that triggers the reset of storage. e,g, 

		
var appVersion = 3;

In custom db, you can add an attribute (data-version="") and (data-expiration="") to control the reset time of the storage.
